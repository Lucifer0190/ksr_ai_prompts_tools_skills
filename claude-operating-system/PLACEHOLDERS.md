# 🔤 Placeholder Reference

> Every project-specific value in this template is written as a `[SQUARE_BRACKET]` placeholder.
> Find-and-replace these across the whole `claude-operating-system/` folder to adapt it to your
> project. Anything still in brackets after setup is something you still need to fill in.

## 🏷️ Project identity

| Placeholder | Meaning | Example |
|---|---|---|
| `[PROJECT_NAME]` | Your product / app name | `Acme`, `Orbit` |
| `[PROJECT_TYPE]` | What kind of software it is | `Multi-tenant SaaS`, `Mobile app`, `CLI tool` |
| `[PROJECT_TAGLINE]` | One-line description of what it does | `Invoicing for freelancers` |
| `[PROJECT_DOCTRINE]` | Your core, non-negotiable product principles | `Privacy-first; no data leaves the device` |

## 🛠️ Tech stack

| Placeholder | Meaning | Example |
|---|---|---|
| `[LANGUAGE]` | Primary language | `TypeScript`, `Python`, `Go` |
| `[FRAMEWORK]` | App / web framework | `Next.js`, `Django`, `Rails` |
| `[FRAMEWORK_VERSION]` | Installed framework version | `16.2.4` |
| `[UI_LIBRARY]` | UI / view library | `React`, `Vue`, `SwiftUI` |
| `[STYLING]` | Styling approach | `Tailwind CSS`, `CSS Modules` |
| `[DATABASE]` | Database / data layer | `Postgres`, `MongoDB`, `SQLite` |
| `[BACKEND_PLATFORM]` | Backend / BaaS platform | `Supabase`, `Firebase`, `custom API` |
| `[AUTH_PROVIDER]` | Authentication mechanism | `Supabase Auth`, `Auth0`, `Clerk` |
| `[AI_PROVIDER]` | AI / LLM provider, if any | `Anthropic Claude`, `OpenAI`, `none` |
| `[HOSTING]` | Where it deploys | `Vercel`, `AWS`, `Fly.io` |
| `[PACKAGE_MANAGER]` | Package manager | `npm`, `pnpm`, `pip` |
| `[TEST_FRAMEWORK]` | Test / e2e tooling | `Playwright`, `Vitest`, `pytest` |
| `[BUILD_CMD]` / `[TEST_CMD]` / `[LINT_CMD]` / `[TYPECHECK_CMD]` | Standard project commands | `npm run build` |
